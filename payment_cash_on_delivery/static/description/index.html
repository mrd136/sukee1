Configuration

Enabled COD Options

After the Installation of this module you will able see the COD payment Acquirer method in Side Setting->Payment->Payment Acquirers .

Here You can Enable/Disabled it's Website Visibility and also select/Create Custom COD Availability Rule.

Inside COD Availability Rule Admin can:

  1. Set Minimum Order Amount to use the COD.
  2. Set Maximum Order Amount to use the COD.
  3. Select Excluded Product from COD.
  4. Enabled/Disable Expected Date of Delivery on Website Product Page.
  5. Enabled/Disable COD Policy of on Website Product Page.
  6. Set Custom Message Content for COD Policy.
  7. Set Custom Message Content to show on Availability of product for COD.
  8. Set Custom Message Content to show on Unavailability of product for COD.


Module Also Provide Country based COD will applicability:

  1. Admin Can Enable Country-wise COD applicability.
  2. Or Enable Country-wise and State-wise COD applicability.
  3. Or Enable Country-wise and ZipCode-wise COD applicability.
  4. Or Even Enable Country-wise, State-wise and ZipCode-wise COD applicability.
  5. At a time Admin Can Enable Multiple Country For COD Applicability.

Website View

COD Availability

During the website visit the user will see the COD Availability Information on the Product View.

Note:
The Expected Date of Delivery Represent the Sum of Current Date and sale_delay set for product.

COD Payment

On the payment page the COD payment option will available and show You have Selected [Cash On Delivery] Payment Option this Sale Order to users message on selection COD Payment Options.
As if the cart contains all those product for which COD is set to Availability by Odoo Admin, then user can Choose COD Payment method.

In case the Cart contain the COD Unavailable product then COD payment will set to disable and display the custom message about COD Unavailability set by Admin for Odoo Back-End on the hover of COD Payment Option
